# 行业简报自动化生成方案

## 概述

一套从**采集 → 筛选 → 生成简报 → 格式转换 → 邮件发送**的全自动行业日报/企业简报生成系统。

- **工作目录**：`/root/.openclaw/workspace/news_daily`
- **数据目录**：`data/{日期}/`（每天独立目录）
- **推送方式**：邮件（SMTP）
- **自动化触发**：通过 `daily_pipeline.py` 串联全流程，可定时调度

## 二、自动化全流程

### 流程图

```
┌─────────────────────────────────────────────────────────────────────┐
│  自动化流程                                                         │
│                                                                     │
│  Step 1 ──→ enterprise_crawler_v3.py   行业资讯采集                │
│             三处室并行，输出 raw/*.json                              │
│                                                                     │
│  Step 1b ─→ step_sentiment_v3.py       搜索舆情采集                │
│             百度/头条搜索                              │
│                                                                     │
│  Step 2 ───→ AI助手（OpenClaw）+ LLM   LLM 筛选 + 生成 md 简报    │
│             读取 config/筛选和生成简报.md 作为提示词                 │
│             读取 raw/*.json 作为候选资讯                              │
│             调用 web_fetch 获取原文                                   │
│             输出 data/{日期}/brief/*.md                              │
│                                                                     │
│  Step 2a ─→ md_to_docx.py              Markdown → Docx 格式转换    │
│                                                                     │
│  Step 3 ───→ step3_send.py             邮件发送                    │
│             遍历 brief/ 目录，.docx 附件 + .md 正文                 │
└─────────────────────────────────────────────────────────────────────┘
```

> **重要**：不再使用 `daily_pipeline.py` 一键编排，每个步骤独立调用。
> **重要**：Step 2（筛选 + 生成 Markdown 简报）通过 AI 助手（OpenClaw）读取 `config/筛选和生成简报.md` 中的提示词、调用 LLM 实现，**不再使用 python 脚本**。

### Step 1 — 行业资讯采集

```bash
python3 scripts/enterprise_crawler_v3.py \
  --date 2026-05-01 \
  --departments 工业互联网处 医药处 原材料_金属_非金属处
```

**工作内容**：
- 读取每个处室的 JSON 配置文件，获取其配置的信息源列表
- 对每个信息源发起网络请求，爬取新闻标题、摘要、发布日期、链接
- 去重、过滤过期内容
- 输出到 `data/{日期}/raw/` 目录

**参数说明**：

| 参数 | 必填 | 说明 |
|------|------|------|
| `--date` | 是 | 目标日期（YYYY-MM-DD），用于生成数据目录和过滤 |
| `--departments` | 是 | 处室名称列表，至少一个。空格分隔。支持：`工业互联网处`、`医药处`、`原材料_金属_非金属处` |
| `--timeout` | 否 | 单次请求超时（秒），默认 600 |

**超时说明**：多来源串行爬取，建议超时 600 秒以上。如网络较慢，可适当增加。

### Step 1b — 舆情搜索采集

```bash
python3 scripts/step_sentiment_v3.py --date 2026-05-01
```

**工作内容**：
- 通过百度/头条等搜索引擎搜索各行业关键词
- 对搜索结果进行 LLM 情感分析（利好/利空/关注）
- 输出到 `data/{日期}/raw/` 目录

**参数说明**：

| 参数 | 必填 | 说明 |
|------|------|------|
| `--date` | 是 | 目标日期（YYYY-MM-DD） |

**超时说明**：舆情搜索涉及搜索引擎访问 + LLM 推理，建议超时 1800 秒。

### Step 2 — AI 筛选 + 生成 Markdown 简报（LLM 方式）

**不再使用 python 脚本**。此步骤由 AI 助手（OpenClaw）执行，过程如下：

1. **读取提示词**：读取 `config/筛选和生成简报.md`，这是系统级指令，定义了筛选标准、简报模板和输出格式。

2. **读取候选资讯**：读取 `data/{日期}/raw/{简报类型}_all_articles.json`，获取采集到的候选资讯列表。

3. **AI 筛选与原文获取**：
   - 按提示词中的筛选标准（最近一周、有参考价值、非股票推荐、不重复等）挑选优质资讯
   - 对选中的资讯，通过 `web_fetch` 工具获取链接原文
   - 结合原文进行摘要提炼和舆情标签分类

4. **生成简报**：参考 `config/example.md` 的样式，按以下章节结构输出：
   - 核心观点摘要（政策/市场/企业三个层面）
   - 行业舆情（资讯+标签）
   - 本地企业舆情
   - 行业影响分析
   - 市场影响分析
   - 企业影响分析

**输出**：`data/{日期}/brief/{简报类型}.md`

> **提示**：简报类型对应处室名称，如 `工业互联网处`、`医药处`、`原材料_金属_非金属处`。需要分别对每个处室的候选资讯生成一份简报。

### Step 2a（可选）— 仅将 Markdown 转为 Docx

```bash
# 方式一：指定单个文件
python3 scripts/md_to_docx.py --src data/2026-05-01/brief/工业互联网处.md --dstdir data/2026-05-01/brief/

# 方式二：默认当前目录
python3 scripts/md_to_docx.py --src example.md
```

**参数说明**：

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--src` | 是 | `example.md` | 输入的 Markdown 文件路径 |
| `--dst` | 否 | 同名 `.docx` | 输出文件名（覆盖默认） |
| `--dstdir` | 否 | 当前目录 | 输出目录 |

**注意**：
- 生成的 `.docx` 遵循铝产业简报样式体系：微软雅黑、深橙/深灰配色、[利好]背景色标签、超链接、页码等。
- **Step 2 输出的仅为 `.md` 文件**，需要运行此脚本转换为 `.docx` 后再发送邮件。
- 多个处室的简报，可以执行多次此脚本逐一转换，也可以用 `for` 循环批量处理。

### Step 3 — 邮件发送

```bash
# 方式一：发送单个 docx 文件
python3 scripts/step3_send.py \
  --brief data/2026-05-01/brief/行业简报_2026-05-01.docx

# 方式二：发送 md + docx 附件
python3 scripts/step3_send.py \
  --brief data/2026-05-01/brief/行业简报_2026-05-01.md \
  --attach data/2026-05-01/brief/行业简报_2026-05-01.docx

# 方式三（推荐）：批量发送整个目录
python3 scripts/step3_send.py \
  --dir data/2026-05-01/brief/ \
  --config config/email.json \
  --date 2026-05-01
```

**参数说明**：

| 参数 | 必填 | 说明 |
|------|------|------|
| `--brief` | 二选一 | 简报文件路径（`.md` 或 `.docx`），与 `--dir` 互斥 |
| `--dir` | 二选一 | 简报目录，批量发送目录下所有文件 |
| `--attach` | 否 | 附件路径（与 `--brief` 配合使用） |
| `--config` | 否 | `config/email.json`（默认） |
| `--date` | 否 | 日期（用于标题和日志目录） |
| `--subject` | 否 | 覆盖邮件主题（默认拼接日期+简报名） |
| `--dry-run` | 否 | 仅打印邮件内容，不实际发送 |

**邮件发送逻辑**：
- `.md` 文件 → 自动转为 HTML 正文
- `.docx` 文件 → 作为附件
- 当 `--brief` 为 `.docx` 时，正文自动生成简短摘要
- 发送日志保存在 `data/{日期}/sent/send_log.json`

## 三、分步骤单独运行

每个步骤独立运行，不再使用 `daily_pipeline.py`。

```bash
# Step 1：行业资讯采集
python3 scripts/enterprise_crawler_v3.py \
  --date 2026-05-01 \
  --departments 工业互联网处 医药处 原材料_金属_非金属处

# Step 1b：舆情搜索采集
python3 scripts/step_sentiment_v3.py --date 2026-05-01

# Step 2：AI 生成 Markdown 简报（在 OpenClaw 中运行，参见上方说明）
#   读取 config/筛选和生成简报.md 作为提示词
#   读取 data/{日期}/raw/*.json 作为候选资讯
#   输出 data/{日期}/brief/*.md

# Step 2a：Markdown → Docx 格式转换
python3 scripts/md_to_docx.py \
  --src data/2026-05-01/brief/工业互联网处.md \
  --dstdir data/2026-05-01/brief/

# Step 3：邮件发送
python3 scripts/step3_send.py \
  --dir data/2026-05-01/brief/ \
  --config config/email.json \
  --date 2026-05-01
```

> 分步运行适合调试或重试某一步失败的情况。

usage: step_sentiment_v3.py [--date DATE]
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `--date` | 是 | 目标日期（YYYY-MM-DD） |

#### `md_to_docx.py`（Step 2a：格式转换）

```
usage: md_to_docx.py [--src FILE] [--dst FILE] [--dstdir DIR]

| 参数 | 必填 | 说明 |

| `--dry-run` | 否 | 仅预览，不发送 |

### 环境变量

| 变量 | 说明 |
|------|------|

## 五、调度方式

### 方式一：Shell 脚本（Linux Cron）

```bash
# 创建一个 shell 脚本 daily.sh
#!/bin/bash
# 行业简报每日自动生成脚本
# 调度：每天 8:00 执行

BASE_DIR="/root/.openclaw/workspace/news_daily"
DATE=$(date +%Y-%m-%d)
LOG_DIR="$BASE_DIR/logs"

mkdir -p "$LOG_DIR"

# Step 1：行业资讯采集
python3 $BASE_DIR/scripts/enterprise_crawler_v3.py \
  --date $DATE \
  --departments 工业互联网处 医药处 原材料_金属_非金属处 \
  >> $LOG_DIR/cron_${DATE}.log 2>&1

# Step 1b：舆情搜索采集
python3 $BASE_DIR/scripts/step_sentiment_v3.py \
  --date $DATE \
  >> $LOG_DIR/cron_${DATE}.log 2>&1
```

```bash
# 每天早上 8:00 执行
0 8 * * * /root/.openclaw/workspace/news_daily/daily.sh
```

> **注意**：Cron 仅调度 Step 1 + Step 1b（采集阶段）。Step 2（AI 筛选生成简报）和后续步骤需人工在 OpenClaw 中触发。
> 如果未来需要全自动，可以将 Step 2~3 也通过脚本实现自动化。

### 推荐执行顺序（人工触发）

| 时间 | 操作 | 预计耗时 |
|------|------|---------|
| 8:00（Cron 自动） | Step 1 + Step 1b：采集启动 | ~30 分钟 |
| 8:30~9:00 | 人工检查采集结果 | — |
| 9:00 | Step 2：AI 生成 Markdown 简报 | ~15 分钟 |
| 9:15 | Step 2a：md → docx 转换 | ~1 分钟 |
| 9:20 | Step 3：邮件发送 | ~1 分钟 |

> 日志目录：`logs/`

## 七、故障排查指南

### 常见错误及解决方法

| 症状 | 可能原因 | 解决方法 |
|------|---------|---------|
| Step 1 超时 | 网络问题或网站拒绝连接 | 检查网络，在命令中加 `--timeout 900` 增加超时 |
| Step 1b 报文包含"API 认证失败"等 | 搜索引擎 API Key 配置异常 | 检查 `step_sentiment_v3.py` 中的 API 配置 |
| Step 2 JSON 解码失败 | 采集结果为空的 JSON | 先单独跑 Step 1 确认采集正常 |
| Step 2 AI 读取不到文件 | 路径或权限错误 | 确认 `data/{日期}/raw/*.json` 存在且可读 |
| Step 3 发送失败 | SMTP 配置错误 | 确认邮箱 SMTP 服务已开启；`password` 为授权码而非登录密码 |
| 无 `.docx` 文件 | Step 2 未正常生成 | 检查 `data/{日期}/brief/` 目录是否存在 `.md` 文件；如需补转，运行 `md_to_docx.py` |

### 调试技巧

1. **分步调试**：先确认 Step 1 输出正常，再执行后续步骤
2. **查看日志**：Cron 日志位于 `logs/cron_YYYYMMDD.log`
3. **邮件干跑**：`--dry-run` 参数预览邮件内容而不发送
