# 行业简报自动化生成方案

## 概述

一套自动化采集、筛选、格式转换、邮件发送的行业日报/企业简报生成系统。
工作目录：`/root/.openclaw/workspace/news_daily`

---

## 一、配置准备（人工完成）

用 `config-parser` skill 将行业需求描述（`.md`）转为标准 JSON 配置文件。

- **输入**：`config/{行业}.md`
- **输出**：`config/{行业}.json`
- **示例**：`config/电子信息.json`

---

## 二、自动化流程

### 步骤1：搜索采集

```bash
python3 scripts/step1_fetch.py --config 电子信息
# 可选指定日期: --date 2026-04-25
```

**4大引擎**：
| 引擎 | 工具 | 用途 |
|------|------|------|
| baidu-search | 百度AI搜索API | 搜索引擎批量搜索 |
| playwright | 无头浏览器 | 百度新闻搜索 |
| web_fetch | requests | 直接抓取指定网页列表 |
| agent-browser | 交互式浏览器 | 需JS渲染/翻页的网站 |

**输出**：`data/{日期}/raw/`，每来源一个 `.txt`，含标题+日期+来源+链接+摘要

### 步骤2：筛选与原文获取（LLM 执行）

1. 读取 raw 目录数据，按 `filter_criteria` 优先级筛选 10 条最有价值条目
2. 逐条用 `web_fetch` 获取原文完整内容（不做删改）
3. 生成含原文的 Markdown 简报
4. **输出**：`data/{日期}/brief/{行业}_brief_{日期}.md`

### 步骤3：格式转换

```bash
python3 scripts/step2_convert.py --brief data/日期/brief/行业_brief_日期.md
# --remove-md    转换后删除原始 .md（默认保留）
# --template 指定模板（默认 config/template_default.json）
# --dir data/日期/brief/   批量转换
```

**输出**：同目录 `.docx` 文件

### 步骤4：邮件发送

```bash
# 发送 .docx（正文预览 + 附件）
python3 scripts/step3_send.py --brief data/日期/brief/行业_brief_日期.docx

# 发送 .md（正文） + .docx（附件）
python3 scripts/step3_send.py --brief xxx.md --attach xxx.docx

# 批量发送整个目录
python3 scripts/step3_send.py --dir data/日期/brief/
```

**邮箱配置**：`config/email.json`
**发送日志**：`data/{日期}/sent/send_log.json`

## 三、完整一键执行

```bash
# 1. 采集
cd /root/.openclaw/workspace/news_daily
python3 scripts/step1_fetch.py --config 电子信息

# 2. 筛选 + 获取原文（由AI人工执行，生成简报 .md）

# 3. 转 Word
python3 scripts/step2_convert.py --brief data/2026-04-26/brief/电子信息_brief_2026-04-26.md

# 4. 发送
python3 scripts/step3_send.py --brief data/2026-04-26/brief/电子信息_brief_2026-04-26.docx
```

## 四、文件结构

```
news_daily/
├── config/
│   ├── 电子信息.json            # 行业配置
│   ├── 电子信息.md              # 行业配置（原始）
│   ├── email.json               # 邮件配置
│   └── template_default.json    # Word模板配置
├── scripts/
│   ├── step1_fetch.py           # 步骤1：多引擎采集
│   ├── _playwright_search.py    # playwright 辅助
│   ├── _agent_browser_fetch.py  # agent-browser 辅助
│   ├── step2_convert.py         # 步骤3：MD→Word转换
│   └── step3_send.py            # 步骤4：邮件发送
└── data/
    └── {日期}/
        ├── raw/                 # 采集原始数据
        ├── brief/               # 简报（.md + .docx）
        └── sent/                # 发送日志
```

## 五、模板配置说明

`config/template_default.json` 支持自定义：
- 页面尺寸、页边距
- 各级标题样式（字体、字号、颜色、对齐）
- 正文、引用、元数据样式
- 分隔线样式
- 页眉页脚文字和样式

可复制为 `template_自定义.json` 使用。
